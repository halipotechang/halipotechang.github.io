MySQL View

MySQL View

用途


  • 由一段select敘述描述而成，呈現的結果即select敘述的查詢結果

  • 簡化複雜的select敘述

  • 隱藏資料，限制程式端工程師能看到的資料

  • 將View設計成唯讀，可避免程式端工程師修改資料


語法:


create view


界習慣會在View名稱前加上V_，以跟Table區分

with [cascaded | local] check option:執行DML時檢查where子句


drop view


View也有可能是其他View的基底資料表， 所以移除後可能造成其他View的錯誤!


alter view


View本身是個不複雜的物件，若要修改View名稱，移除後重新建立即可


可安全異動資料的View

  • 須符合以下條件
    1. 無計算欄位、函數欄位
    2. 基底資料表只有一個
