JAVA 例外處理

JAVA 例外處理

Exception物件


  • Java程式執行時，如果發生異常狀況，可借助例外處理
    • 程式發生(產生)例外時，視為產生了一個Exception物件
  • 例外處理可將一般正常處理程序與錯誤處理程序做到分開敘述，讓程式可讀性提高

分類

  • 執行時期的例外
    • 這一類的例外，不一定要處理 .也稱為不必檢查的例外 (unchecked exceptions)
    • 例如 Runtime Exception 及其子類別
  • 非執行時期的例外
    • 這一類的例外，一定要處理 .也稱為必須檢查的例外 (checked exceptions)
    • 例如 IOException、SQLException…等

try

  • try { }:將正常處理程序的程式碼置於try程式區塊中

catch

  • catch (MyException e) { }:當捕捉到例外物件時將錯誤處理程序的程式碼置於catch的程式區塊中
  • 類別位階高者需置於類別位階低者的後面

finally

  • finally { }:將一定要執行的程式碼放在 finally的程式區塊裡
  • 無論發生什麼情況皆會執行的程式區塊，可用來釋放有限的資源，例如關閉(close)資料庫連
    線、檔案讀取等

範例



throws

  • 方法定義時，可使用 throws 關鍵字將可能發生的例外，丟出給呼叫此方法的程式去處理

  • 對於checked exceptions，在呼叫有 throws 關鍵字的方法時，必須將該方法置於:

    1. 將該方法置於 try { } 程式區塊中
    2. 將該方法置於定義有 throws 關鍵字的方法中

  • 子類別覆寫其父類別定義有 throws 的方法時，子類別所 throws 的 Exception 必須與父類別被覆寫方 法的 Exception 一樣或是更低階


throw

  • 可使用 throw 關鍵字，將方法內的例外手動丟出

    • 該物件必須是 java.lang.Throwable 類別的子類別
    • throw 「一個可被丟出的物件」

範例


自訂例外

  • 必須繼承 Throwable 或 Exception 或 RuntimeException

  • 通常會包含兩個建構子

  • 程式內一樣可利用 throw 關鍵字，將例外拋給負責處理此例外的 catch { } 區塊處理


斷言(Assertion)機制

  • 用來維護程式使之更堅固(robust)，零錯誤
  • Assertion通常用來檢查一些關鍵的值，避免這些值有錯誤時，讓程式無法繼續執行
