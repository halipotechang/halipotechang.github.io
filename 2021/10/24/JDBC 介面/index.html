JDBC 介面

JDBC 介面

Connection 介面


  • Connection代表資料庫一個連線通路，傳遞一系列的SQL指令給資料庫，並管理這些指令的認可與中止

  • Connection非常珍貴，使用完畢後務必在finally{ }完成歸還的動作!


資料庫位置 (URL)

  • 資料庫URL格式: jdbc:<Subprotocol>:<Subname>

    • jdbc:mysql://localhost:3306/HR

    MySQL 8 以後的驅動版本需在URL後接著?serverTimezone=Asia/Taipei

關閉連線 – close()

對con檢查不為null是為了避免連線尚未建立前就發生例外


Statement 介面 (靜態SQL指令)

• 藉由Connection的方法:Statement createStatement()來建立與取得Statement物件

  • 執行SQL指令的主要兩個方法

    1. ResultSet executeQuery(String sql):查詢資料庫，傳回ResultSet
    2. int executeUpdate(String sql):更新資料庫，傳回成功更新的筆數

PreparedStatement 介面


  • PreparedStatement為Statement的子介面，因此也可以執行SQL指令

  • 藉由Connection的方法: PreparedStatement prepareStatement(String sql)

    資料庫 可預先編譯SQL指令，執行效能較快，常用於需變數傳遞且重複執行的SQL指令

可對SQL指令裡，未知或是需動態改變的參數設定為 “ ? “，並在每次執行時，對 ? 置入 不同的值

setType(int idx, Type value) ? 的索引值是從1開始

  • 執行SQL指令的主要兩個方法

    1. ResultSet executeQuery(String sql):查詢資料庫，傳回ResultSet
    2. int executeUpdate(String sql):更新資料庫，傳回成功更新的筆數

CallableStatement 介面 (預存程序)

  • CallableStatement prepareCall(String sql):

    使用預存程序(stored procedure)，預存程序已事先內建在資料庫中，通常比預先編譯的效能佳

  • 程式設計師只需知道預存程序的名稱與輸入輸出的參數，無需暸解SQL指令

  • 不同的資料庫有不同的預存程序語法;對三階開發人員來說，意義不大


ResultSet介面

  • 當執行executeQuery()方法，該方法即傳回從資料庫查詢的結果資料列

  • 移動資料列游標的相關方法

    • next()、previous()、first()、last()、beforeFirst()、afterLast()
    • next()傳回boolean值

      部份移動游標方法為JDBC 2.0時新增，需在建立Statement時設定

  • 取出查詢結果資料欄的方法

    1. type getType(int columnIndex)
    2. type getType(String columnName)

      index從1開始，一般來說使用index較有效率

  • 其它方法

    • int getRow():取得目前的列編號
